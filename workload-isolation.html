Atlas Cluster

Atlas Search
Lucene-based full-text and vector search running on dedicated mongot processes — completely isolated from CRUD.
Search Nodes
Search indexes managed by mongot process — not the mongod that handles your writes
Supports fuzzy match, semantic (vector), faceted, and hybrid search
Search tier can scale independently from your database tier
No separate Elasticsearch cluster to sync and maintain

The Architecture Transformation

❌ Before Atlas Workload Isolation

🔴 Analytics queries crash production during peak hours
🔴 Nightly ETL to warehouse — analysts work with day-old data
🔴 Separate Elasticsearch cluster to sync and maintain
🔴 Over-provisioned primary cluster to absorb all workloads
🔴 Three teams, three platforms, three data copies
🔴 SLA violations when scheduled reports run

✅ After Atlas Workload Isolation

🟢 Analytics nodes absorb all heavy reads — OLTP unaffected
🟢 Analysts query live data with sub-second replication lag
🟢 Atlas Search nodes replace Elasticsearch entirely
🟢 Each tier sized and billed independently
🟢 One Atlas cluster, one driver, one data model
🟢 Guaranteed SLA with resource isolation by design
